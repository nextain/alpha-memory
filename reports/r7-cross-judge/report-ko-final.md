## 요약

R7 교차 판정 평가 결과, **벤치마크 자체에 근본적 구조적 결함**이 있어 현재 점수로 메모리 시스템 순위를 매기는 것은 신뢰할 수 없음이 밝혀졌습니다. 2-judge 합의 시스템은 KO에서 성공적으로 자의성을 줄였으나(95-99% 일치), EN 결과는 GLM의 극단적 관대함이 원인이 된 치명적인 판정관 불일치(14-84%)를 드러냈습니다. 가장 중요하게, **airi — 메모리가 전혀 없는 시스템 — 이 EN에서 41.6%를 기록**했으며, 이는 벤치마크가 실제 메모리 검색보다 LLM 사전 학습 지식을 보상하고 있음을 증명합니다.

### 핵심 발견

1. **KO 점수는 보편적으로 처참** (14-36%) — 두 판정관 모두 한국어 메모리 검색이 전반적으로 실패함을 정확히 식별 (95-99% 일치)
2. **EN 판정관 불일치 극심** — GLM은 우수 시스템에 80%+ PASS, Gemini는 ~10%만 PASS. 합의는 중재자로 해결하나 양 극단 모두 문제
3. **airi (메모리 없음) EN 41.6%** — 영구 메모리가 없는 시스템이 메모리 벤치마크에서 ~0%여야 함. 이 단일 사실이 현재 EN 채점을 무효화
4. **sillytavern EN 91.5%** — 단순 키워드 저장소가 모든 목적형 아키텍처를 능가하는 것은 벤치마크 게임의 명확한 신호
5. **모든 시스템이 F(abstention) 등급** — abstention 페널티 규칙이 정직한 거부보다 환각을 장려

**분석**: 한국어 메모리 성능은 보편적으로 참혹합니다. 1위인 letta (35.8%)마저 항목의 거의 3분의 2를 실패합니다. 높은 판정관 일치율(95-99%)은 사실 **바닥 효과(floor effect)**입니다 — 응답이 명백히 틀리거나 비어 있어 두 판정관이 쉽게 FAIL에 동의합니다. 이 일치율은 채점의 타당성을 검증하는 것이 아니라, 실패가 모호하지 않음을 확인할 뿐입니다.

주목: naia (25.2%)와 mem0 (24.5%)는 KO에서 기능적으로 동일하며, 이는 두 시스템이 동일한 mem0 파이프라인을 사용하는 것으로 알려진 mem0 KO dedup 버그와 일치합니다.

## 2. 근원 분석

### 2.1 KO 일치율 99% vs EN 14-84%인 이유

**KO 바닥 효과**: 한국어 메모리 검색이 너무 나빠서 두 판정관이 쉽게 FAIL에 동의합니다. 시스템이 팩트 쿼리의 80%에 "기억에 없습니다"를 반환할 때, 판정은 명백합니다. 보편적으로 처참한 점수에 대한 높은 일치율은 의미가 없습니다 — 이는 불일치의 상한선이지, 품질의 하한선이 아닙니다.

**EN 의미적 모호성**: 영어 응답은 유창하고 그럴듯하지만 사실적으로 틀린 경우가 많습니다. GLM은 의미적 근접성("충분히 가깝다" 매칭)을 보상하는 반면, Gemini는 엄격한 사실적 정확도를 요구합니다. 이는 응답이 "명백히 정확함"과 "명백히 틀림" 사이의 모호한 영역에 있을 때 대규모 불일치를 만듭니다.

### 2.2 airi 역설 (재검토)

airi는 메모리 시스템이 없습니다 — LLM 사전 학습에서만 응답을 생성합니다. R7 EN에서 71% 판정관 일치율로 41.6%를 기록합니다. 카테고리별 분해:

| 카테고리 | airi EN | 이유 |
|----------|---------|------|
| abstention | 100% | 알 수 없는 팩트에 "모른다"고 정확히 말함 |
| irrelevant_isolation | 100% | 유출할 개인정보가 없음 |
| contradiction_direct | 100% | 사전 학습으로 논리 처리 |
| contradiction_indirect | 67% | 사전 학습 추론 |
| noise_resilience | 75% | 사전 학습으로 노이즈 필터링 |
| unchanged_persistence | 67% | 올바르게 추측 |
| proactive_recall | 25% | 능동적으로 제안할 수 없음 |
| direct_recall | 0% | 특정 팩트 회상 불가 |
| semantic_search | 0% | 존재하지 않는 메모리 검색 불가 |
| temporal | 0% | 시간적 지식 없음 |

**airi의 41.6%는 메모리 검색이 필요 없는 카테고리에서 전부 나옵니다** (abstention, isolation, 논리, 노이즈 필터링). 이는 벤치마크가 "일반 지능"과 "메모리 능력"을 혼동하고 있음을 증명합니다. 타당한 메모리 벤치마크라면 메모리 없는 기준선에 ~0%를 주어야 합니다.

### 2.3 GLM 관대함 vs Gemini 엄격함

| 행동 | GLM-5.1 | Gemini 3.1 Pro |
|------|---------|----------------|
| 의미적 근접성 | "충분히 가깝다" 수용 | 정확한 일치 요구 |
| 환각 허용도 | 높음 (유창성 보상) | 낮음 (추측 처벌) |
| 키워드 매칭 | 느슨함 (유의어 자유 수용) | 엄격함 (핵심 개념 필요) |
| 기준선 (airi) 처리 | 100/240 PASS | 34/240 PASS |

**어느 판정관도 단독으로는 정확하지 않습니다.** GLM은 너무 관대하고(환각 수용으로 점수 부풀림), Gemini는 너무 엄격합니다(타당한 의미적 매치 거부). 진실은 그 사이에 있습니다.

### 2.4 Abstention F등급 구조적 결함

EN에서 8개 시스템 모두 F(abstention)를 받습니다. 이유:
1. 메모리 시스템이 모든 질문에 답하려 함 → 일부 abstention 쿼리에 답변 (거부하지 않아 FAIL)
2. 이진 "abstention 하나라도 실패하면 F" 규칙은 실수 하나 = 등급 F를 의미
3. 이는 시스템이 불확실할 때 항상 거부하도록 장려하는데, 이는 역설적으로 회상 점수를 해칩니다

abstention F등급은 신뢰도 보정과 메모리 품질을 혼동합니다. 시스템이 완벽한 회상을 가지더라도 abstention 쿼리 20개 중 1개에 답하면 F를 받을 수 있습니다.

## 3. 개선 권고사항

### 3.1 판정관 시스템 개선

1. **기준선 고정(Baseline Anchoring)**: 매 평가에 자동으로 airi(메모리 없음)를 주입. airi가 5% 초과 점수를 받으면 판정 프롬프트가 무효이며 강화 필요. 객관적 보정 기준점을 만듦.

2. **구조화된 출력**: 자유형 PASS/FAIL 텍스트를 JSON 출력으로 교체: `{"verdict":"PASS","confidence":0.9,"matched_keywords":["서연"],"reason":"..."}`. 파싱 모호성을 제거하고 신뢰도 가중 채점을 가능하게 함.

3. **0-3점 척도**: 이진 PASS/FAIL을 다음으로 교체:
   - 0: 환각 (메모리에 없는 정보 날조)
   - 1: Abstention ("모른다" — 중립, 벌점 없음)
   - 2: 부분 일치 (올바른 주제이지만 잘못된/누락된 세부사항)
   - 3: 정확한 팩트 일치 (메모리 저장소에서 정확한 회상)

4. **이중 판정 + 인간 검토**: 판정관이 불일치하는 항목은 편향된 제3의 LLM 의견에 의존하지 말고 인간 검토용으로 플래그 지정.

### 3.2 벤치마크 설계 개선

1. **합성/허구 팩트**: 실제 팩트를 허구로 교체 (예: "내 여동생이 가장 좋아하는 과일은 두리안이야", "나는 넥슨 17층에서 일해"). 이것은 LLM이 사전 학습 지식을 사용할 수 없게 보장하여 airi 기준선을 즉시 ~0%로 끌어내림.

2. **방해(Distractor) 메모리**: 대상 팩트와 함께 유사하지만 충돌하는 메모리를 주입. 시스템이 올바른 메모리를 검색하는지 테스트. 예: "여동생이 사과를 좋아해"와 "여동생이 두리안을 좋아해"를 모두 저장 — 시간적 맥락에 기반한 올바른 것만 보상.

3. **메모리 격리 평가**: 벤치마크를 두 점수로 분리:
   - **메모리 회상 점수**: 저장된 메모리에서 팩트 검색이 필요한 카테고리만 (direct_recall, semantic_search, temporal, entity_disambiguation, multi_fact_synthesis)
   - **추론 점수**: 논리/행동을 테스트하는 카테고리 (contradiction, noise_resilience, abstention, irrelevant_isolation)
   - 둘을 따로 보고; 혼동 금지

4. **고유 식별자**: 저장된 팩트에 무작위 ID나 특정 숫자 포함 (예: "내 사번은 8472931이야"). 쿼리는 정확한 ID를 검색해야 함. 퍼지 매칭으로 점수 획득을 방지.

### 3.3 채점 방법론 개선

1. **Abstention F등급 제거**: 페널티 시스템으로 교체:
   - 정확한 회상: +1
   - 환각/오답: -0.5 (추측 처벌)
   - Abstention: 0 (중립 — 정직한 거부를 처벌하지 않음)

2. **신뢰도 가중 채점**: 응답 LLM이 낮은 신뢰도를 표현하면("아마...일 것 같은데") 보상을 감소. 높은 신뢰도의 오답은 불확실한 오답보다 더 큰 페널티를 받아야 함.

3. **카테고리 가중치 재조정**: 현재 12개 카테고리가 동일 가중치. 메모리 의존 카테고리(direct_recall, semantic_search, temporal)의 가중치를 높이고, 메모리가 필요 없는 추론 카테고리의 가중치를 낮춤.

### 3.4 naia 우선 개선사항

결과를 바탕으로 naia가 우선시해야 할 사항:

1. **LocalAdapter 벡터 검색 수정 (alpha-memory#5, Critical)**: naia KO는 현재 25.2% — mem0(24.5%)과 본질적으로 동일. 이유는 두 시스템이 동일한 mem0 백엔드를 사용하기 때문. LocalAdapter의 벡터 검색은 한 번도 작동한 적이 없음. 수정이 단일 최고 영향 개선사항.

2. **한국어 언어 파이프라인**: naia KO 25.2% vs EN 83.3% = 58pp 격차. 영어 최적화 LLM dedup/정규화 파이프라인이 한국어 텍스트를 파괴함. 한국어 인식 처리를 갖춘 LocalAdapter로 전환 필요.

3. **Abstention 보정**: naia abstention 100% KO / 55% EN. 100% KO는 위양성 — 아무것도 검색되지 않아 LLM이 "모르겠다"로 기본 설정. 실제 수정은 먼저 벡터 검색(#5)이 필요하고, 그 다음 코사인 유사도 임계값 조정.

4. **불변 지속성 (alpha-memory#10)**: naia EN 73%, KO 33%. 모순 업데이트가 관련 없는 팩트를 연쇄 삭제. 직접 15-20pp 손실을 초래하는 알려진 버그.

5. **시간 메모리 (alpha-memory#8)**: naia EN 80%, KO 20%. 모순 업데이트 시 팩트가 덮어씌워져 이력 상실. append-only 시간 저장 필요.

## 7. 결론

R7 교차 판정 평가는 벤치마크의 핵심 문제를 성공적으로 식별했습니다:

1. **벤치마크는 메모리가 아닌 일반 지능을 측정**: airi(메모리 없음)가 41.6%를 기록한 것은 추론/논리 카테고리가 메모리 능력과 무관하게 점수를 부풀린다는 증거
2. **판정관 관대함이 EN 점수를 지배**: GLM의 의미적 근접성 매칭이 사실적 회상 정확도를 반영하지 않는 부풀려진 점수를 생성
3. **KO는 진정한 실패 모드**: 보편적으로 처참한 한국어 점수에 대한 95-99% 일치율이 가장 신뢰할 수 있는 발견 — 한국어 메모리 검색은 어디서나 망가져 있음
4. **Abstention F등급은 역효과**: 정직한 거부를 처벌하여 환각을 장려

**다음 단계**: 합성 팩트뱅크, 방해 메모리, 기준선 고정 구현 (R8). 이 변경으로 airi를 ~0%로 끌어내리고 메모리 시스템 간 의미 있는 차별화를 만들 수 있어야 합니다.
